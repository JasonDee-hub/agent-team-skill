# 域外任务 Grilling 协议

用于不满足低风险短任务豁免的域外知识工作。主控亲自完成追问与共识确认后，才派 Atlas。

## 豁免判断

同时满足以下条件可免 grilling，直接使用已有上下文派 Atlas：

- 交付物短小、一次性且容易修改
- 用户目标、受众和格式已足够明确
- 不涉及医疗、法律、财务等高风险决策
- 不会直接执行发布、发送、购买等外部动作

若只缺一个会阻塞交付的信息，最多补问一题；否则进入完整 grilling。

## 题量与进度

- 总题量最多 5 问，常见 3–4 问；只问会改变交付的分叉。
- 第 1 问先确定交付类型或成功标准。答完后锁定总题数并告知进度，此后只能减问，不能加问。
- 第 2 问起标记 `问题 k/N`。用户说「剩余按推荐」「你定」或「够了」时，用推荐默认收束。
- 能从对话或材料推断的信息直接采用，不占题额。到上限仍有歧义时采用推荐默认，不再追问。

## 提问方式

- 一次只问一个问题，等待用户回答后再继续。
- 每题提供 2–4 个具体选项，标出 `★ 推荐` 并给一句理由；允许用户自定义答案。
- 有结构化提问工具时优先使用。决策顺序通常是：交付类型 → 受众/用途 → 材料与约束 → 篇幅/结构。
- 结束时用 5–8 条要点复述交付物、不交付内容和关键约束，获得确认或默认认可后再派 Atlas。
- 禁止用「简要/标准/详尽」三档菜单代替需求澄清，禁止无选项空问和无上限追问。

## 示例

```text
问题 1/？（答完后给出本轮总题数）
这次「小说」要哪种交付？

A) 可独立阅读的短篇（约 3000–5000 字）★ 推荐：范围清晰，可一次成稿
B) 长篇开篇 + 大纲
C) 中篇（约 1–2 万字）
D) 连载第一章

回 A/B/C/D，或「其他：…」
```

用户选 A 后：

```text
已记录：短篇。本轮共 4 问；已完成 1，还剩 3。
随时可说「剩余按推荐」。

问题 2/4：主要读者是谁？
…
```

## 派遣与交付

- 只派 `万事通 Atlas（非开发主场，通才产出）`；逐一说明为何不用其他角色。
- 花名保持「万事通 Atlas」，不得改成专职头衔。
- 将共识要点写入派遣包 `context` / `acceptance`。
- 默认一次成稿；仅当共识明确要求或篇幅、风险较高时，先结构稿再成稿。
- 交付时说明由通才 Atlas 兜底，并非该领域专职专家。
